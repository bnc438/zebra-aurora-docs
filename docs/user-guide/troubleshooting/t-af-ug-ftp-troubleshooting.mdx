## Firewall Settings

If the FTP server is not receiving messages, review the firewall settings. Ensure that the FTP port (default: 21) is open and that the firewall allows both inbound and outbound FTP traffic.

## Invalid Folder Attributes

In cases where the FTP server application cannot write on the destination folder, select the folder and use the **ALT+ENTER** command to open the properties window and verify that the **Read-only** checkbox is not enabled.

## Invalid Permissions

The FTP server configuration prevents certain operations. If the log of the FTP server application shows a critical file transfer error, ensure that the FTP user has permissions to create directories and upload files. Activate all file operation options (read, write, delete, append, create directory) in the FTP server configuration.
