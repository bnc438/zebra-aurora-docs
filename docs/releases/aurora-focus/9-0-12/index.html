Zebra Aurora Focus™ MV/FIS Configuration Utility

Description

Product Description Zebra Aurora Focus™ is a host-side application which allows for the configuration Zebra FS10, xS20, xS40, xS70, FS42, NS42 and FS80 devices. Zebra Aurora Focus™ Deep Learning Editor is companion software for training the Deep Learning Anomaly Detection tool. To use the editor, a physical or emulated NS42 device licensed for Anomaly Detection is required. It is installed independently and available in two versions: CPU and GPU versions. The GPU version is to train models using NVIDIA GPUs.

Known Issues​

  • Under some conditions, Datamatrix performance issues have been observed with Barcode tool.
    • Workaround: Use the Datacode tool.
  • Under some conditions, barcode decode performance may be impacted.
    • Workarounds:
      • Disable unused symbologies.
      • Disable configuration barcodes (if not used) from the General menu.
      • Set expected module size appropriately.
  • On occasion, dual update option on firmware update fails.
    • Workaround: Update firmware without using dual update. Then repeat the same firmware version update using "Force Update" with "Dual Update" checked.
  • Sometimes HMI, after removing a license, does not update the screen.
    • Workaround: Refresh or switch tabs to update the screen.
  • On some devices, USB port speed may not change from 2.0 to 3.0 when updated in Aurora.
    • Workaround:
      • Switch from 3.0 to 2.0 and apply settings.
      • Switch from 2.0 to 3.0 and apply settings.
      • Reboot the device.
  • After very long idle durations, the JavaScript editor window may lose connection.
    • Workaround: Restart Aurora Focus.
  • Autotune may fail if the center of image is black (no features) and the barcode is outside the center.
    • Workaround: Run autotune with the barcode closer to the center of field of view.
  • FS80 currently does not support IP fallback to a default 169.254.x.x network IP.
    • Workaround: Use with a DHCP server.
  • Reboot is required to see the Anomaly Detection tool in Aurora Focus after the device upgrade license is applied.
  • Rarely, the job list in Aurora Focus does not show a job after reboot.
    • Workaround: Click Refresh at the top of the job list.
  • Rarely, it may take a few seconds for the UI to update after selecting or deselecting all symbologies.
  • When adding a very large number of tools to the VS flow builder, ROI boundaries may not be visible or performance issues may occur.
  • When using Connectivity Gateway and configuring more than one GPIO, changes may not take effect immediately.
    • Workaround: Restart Runtime.
  • WebHMI objects may not always load properly when opening the WebHMI page.
    • Workaround: Refresh the page.
  • WebHMI objects may not always load properly on reboot.
    • Workaround: Refresh the page.
  • If the device is factory reset over USB, or where NTP is not available, the device real-time clock may be reset.
    • Workaround: Open WebHMI and navigate to the license page to sync time with the browser host, or set the time manually.
  • When saving images over FTP using a hostname, transfer delays may occur and the image is not transferred.
    • Workaround: Use the FTP server IP address instead of hostname.
  • When using the Object Plus tools, operation may fail due to contrast value.
    • Workaround: Adjust contrast until it succeeds and the result is satisfactory.
  • Rarely, OCR performance may be impacted depending on the inspected character set.
    • Workaround: Adjust ROI size.
  • OCR performance is sometimes impacted when ROI is close to one or more characters.
    • Workaround: Increase ROI slightly.
  • Some ZETI commands do not work with the shortened command format.
    • Workaround: Use the extended version of the ZETI command.
  • In some cases, using many image banks causes slight processing delays.
    • Workaround: Reduce the number of image banks if possible.
  • Device may not be discoverable in Aurora Focus when starting firmware update in HMI until update completes and the device reboots.
    • Workaround: Wait for firmware update completion.
  • Firmware updates fail if memory consumption is already very high.
    • Workaround: Reboot the device before performing firmware update.
  • Uninstalling InstallShield does not always remove all registry entries.
    • Workaround: Use regedit and manually delete the Aurora Focus folder under Computer\HKEY_LOCAL_MACHINE\SOFTWARE\Zebra.
